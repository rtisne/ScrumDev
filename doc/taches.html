Liste des tâches :

  • 1) Création de la base de données :
    - Créer les tables User Story, Sprint, Tâche, DépendanceTaches, Utilisateur,Projet, TestE2E (voir annexe)
    - Créer les relations entre les tables

  • 2) Codage des requêtes de gestion des comptes utilisateur :
    - Créer la requête permettant d’ajouter un utilisateur dans la base
    - Créer la requête permettant de récupérer la liste de tous les utilisateurs
    - Créer une requête de suppression d’un utilisateur.
    - Créer une requête de mise à jour des données d’un utilisateur
    - Créer la requête permettant de vérifier qu’un utilisateur est dans la base
    - Créer la requête permettant d’afficher toute les données d’un utilisateur
    - Créer la requête permettant de récupérer un utilisateur selon son nom ou son email

  • 3) Codage des requêtes permettant la gestion des projet
    - Créer la requête permettant d’afficher la liste de tous les projets d’un utilisateur donné
    - Créer la requête permettant d’ajouter un nouveau projet dans la base
    - Créer la requête permettant de récupérer la liste des User Story d’un projet en particulier
    - Créer la requête permettant d'ajouter une User Story
    - Créer la requête permettant de faire la liste des sprints d’un projet en particulier
    - Créer la requête permettant d’ajouter une tâche pour un projet
    - Créer la requête permettant de faire la liste des tâches d’un projet

  • 4) Création d’une structure pour les pages de connexion (Template) :
    - Création d’une page basique avec avec une entête et un titre (voir page Inscription)
    - Création d’un bouton “Inscription” permettant d’aller sur la page d'inscription
    - Création d’un bouton “Connexion” permettant d’aller sur la page de connexion

  • 5) Création de la page d'inscription
    - Utilisation du template de la tâche 3
    - Création d’un formulaire comprenant 4 champs “Nom, prénom,email et password”
    - Création d’un bouton permettant de valider l'inscription utilise les données du formulaire et la requête d’ajout d’un utilisateur dans la base

  • 6) Création de la page de connexion
    - Création d’un formulaire avec deux champs “email,password”
    - Création d’un bouton permettant de valider la connexion
    - Utilisation de la requête permettant de vérifier qu’un utilisateur est dans la base

  • 7) Création de la page de la page “monCompte”
    - Création d’un formulaire avec 4 champs “Nom,prénom,email,et password”
    - Affichage des données de l’utilisateur dans le formulaire avec la requête d’affichage des données
    - Création d’un bouton “modifier” permettant de valider la modification en utilisant les données du formulaires et la requête de mise à jour des données d’un utilisateur

  • 8) Création de la page “ListeDesProjets”
    - Récupération de la liste des projets d’un utilisateur à l’aide de la requête associés
    - Affichage des données dans une liste au centre de la page
    - Création d’un bouton permettant d’aller sur la page de création d’un projet
    - Ajout d’un bouton permettant de modifier les données d’un projet dans la liste si l’utilisateur est le créateur du projet
    - Permettre qu’un projet de la liste soit cliquable afin d’amener vers la page “home” du projet en question

  • 9) Création de la page “creerProjet”
    - Création d’un champs texte “nom” pour le nom du projet
    - Création d’une zone de texte pour la description du projet
    - Création d’une date sélectionnable pour la date de livrable du projet
    - Création d’une liste des membres participant au projet
    - Création d’un bouton “ajouter” permettant d’ajouter un membre à la liste (l’utilisateur doit être dans la base de données, utilisation de la requête permettant de récupérer un utilisateur selon son nom ou son email)
    - Création d’un champ texte “Product Owner”
    - Création d’un bouton “créer” permettant de créer le projet avec les informations du formulaire (utilisation de la requête de création d’un projet”

  • 10) Création d’une structure pour les pages de gestion d’un projet (Template) :
    - Création d’un header avec plusieurs onglets “Home,Backlog,Sprints,Graphs et Configuration”
    - Affichage du titre du projet en haut à gauche de la page
    - Utilisation du bouton permettant de revenir au informations relative à un compte utilisateur

  • 11) Création de la page “Home” pour un projet donné
    - Affichage de la description du projet
    - Affichage du sprint en cours de réalisation
    - Affichage du graphique Burn-Down-Chart

  • 12) Création de la page “Backlog” pour un projet donné
    - Création d’un tableau permettant d’afficher les User Story relative à un projet
    - Utilisation de la requête permettant de récupérer la liste des User Story
    - Création d’un bouton permettant de créer une User Story en allant vers la popup de création d’une User Story

  • 12) Création de la page “Backlog” pour un projet donné
    - Création d’un tableau permettant d’afficher les User Story relative à un projet
    - Utilisation de la requête permettant de récupérer la liste des User Story
    - Création d’un bouton permettant de créer une User Story en allant vers la popup de création d’une User Story

  • 13) Création de la pop-up de création d’une User Story
    - Création d’un champs texte “Titre” pour le titre de la User Story
    - Création d’une zone de texte “Description” pour la description de la User Story
    - Création d’un champs pour le coût
    - Création d’un bouton “valider” permettant d’ajouter la user story dans la base ( utilisation de la requête d’ajout d’une user story)

  • 14) Création de la page “Sprint” pour un projet donné

    - Chaque sprint de la liste doit pouvoir être cliquable afin d’aller sur la page de son kanban respectif

  • 15) Création de la page “KanBan” pour un sprint en particulier d’un projet
    - Création d’un bouton permettant de créer une tâche en allant vers la popup de création d’une tâche

  • 16) Création de la pop-up de création d’une tâche

    - Utilisation de la requête permettant de récupérer la liste des membre du projet
    - Création d’une liste déroulante avec la liste des tâches du projet

  • 18) Création de la page de “Configuration” pour un projet donné
